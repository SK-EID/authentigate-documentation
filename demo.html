Testing

Authentigate offers two options for testing in the demo environment:

  1. Explore the user experience (UX/UI)

  2. Integrate your system for technical testing

Option 1: View UX and UI in Demo Environment

Try out the Authentigate flow and user experience directly in your browser — no registration needed.

How to Try It

  1. Open the demo client page: https://oidc.demo.sk.ee/sample-rp/

  2. Set the parameters on the demo page:

    1. Scope – select which attributes to request

    2. Age comparator – used when age_over or age_under are selected

    3. Language – choose your preferred language

    4. Demo client – select the client setup to simulate:

      1. alco123 – test age verification (e.g., for alcohol purchase)

      2. bank321 – test authentication (e.g., for logging in)

  3. Choose an authentication method:

    1. Smart-ID Demo Accounts

      Example
           Personal number (Lithuania): `50001029996`

    2. Mobile-ID Demo Accounts

      Example
           Mobile: `+37200000766`
     Personal number: `60001019906`

  4. Start the authentication process. When the verification code appears, wait a few seconds — after successful authentication, you’ll be redirected to the demo client page.

You can also create your own Smart-ID demo account if needed.

Option 2: Test Integration with Demo Environment

If you want to test how Authentigate works within your own system setup, we support demo integrations without needing a signed contract.

Steps to Get Started

  1. Request demo access via our e-Service Portal

  2. Select the "Demo Plan" (free of charge).

  3. In the order form, include your test environment details. See section Client Registration in the Authentigate documentation.

  4. Once your access is configured, you’ll receive a confirmation.

  5. Set up the integration between your test system and our demo environment.

  6. Start testing using section API Specifications.
